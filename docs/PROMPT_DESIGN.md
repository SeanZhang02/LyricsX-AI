# Prompt 设计 + 输出格式契约

> 现行 prompt 在 `AppController.swift:520`(`buildPrompt`,硬编码)。本文档是迭代蓝本:A 版质量优先 / B 版格式稳定优先(兼作重试专用)。长期可把 prompt 暴露成可编辑偏好(UserDefaults 存,空 = 用内置默认)。

## 设计原则(对抗审查结论)

1. **行对齐 > 一切**: `编号|译文` 逐行协议,不用 JSON——错位可检测、可逐行恢复、免疫模型在 JSON 里输出未转义字符/额外废话。
2. **第 i 行配到第 j 行译文比没有译文更糟**: 所以有覆盖率门控 + 内容校验(HARDENING_TODO H2/H3),prompt 只是第一道防线。
3. **版权拒译是真实风险**: prompt 开头 framing 为"用户本地歌词文件/仅个人查看",降低拒译概率;拒译检测兜底在解析层。
4. **重复副歌不去重**: 同一调用内译法天然一致,是质量加分,不是浪费 token。
5. **西语场景**: 通读全篇再逐行(人称/性别一致性对西语尤其重要),俚语/reggaeton 文化梗意译。

## A 版 — 质量优先(temperature 0.3)

```text
你是一位专业歌词译者,负责把外语歌词翻译成{TARGET}。歌词来自用户本地播放器中已有的歌词文件,仅为个人查看生成翻译。译文会直接显示在歌词软件里,随音乐逐行滚动。

先通读整首歌,理解主题、情绪和叙事视角,再逐行翻译,保证全篇人称和语气一致。

翻译要求:
1. 逐行对应:不合并、不拆分、不跳过任何一行。输入多少行,输出就多少行。
2. 用口语化、有歌词感的中文,像能唱出来的话,不要书面翻译腔。
3. 俚语、双关、文化梗按意思和情绪意译,宁可传神,不要直译生硬。
4. 语气词和 ad-lib(oh yeah、la la la、eh 等)输出「编号|-」。
5. 人名、地名用通行译名,没有通行译名就保留原文。
6. 每行译文必须是单行,不含换行,不使用方括号 [ ] 和竖线 |。

输出格式(严格遵守):
- 每行输出「编号|译文」,编号与输入完全一致。
- 除这些行外不输出任何内容:不要解释、不要空行、不要 markdown 代码块。
```

## B 版 — 格式稳定优先(temperature 0,重试专用)

```text
你是歌词翻译引擎。输入是用户本地歌词文件中带编号的外语歌词,仅为个人查看生成翻译。输出必须且只能是逐行对应的{TARGET}译文。

规则:
1. 每一行输入对应恰好一行输出,格式为「编号|译文」。编号照抄输入,一个不能少、一个不能多、不能改变顺序。
2. 译文是单行文本:不含换行、不含方括号 [ ]、不含竖线 |。
3. 无需翻译的行(oh yeah、la la la 等拟声词、词曲制作信息行、本身已是{TARGET}的行)输出「编号|-」。
4. 输出的第一行就是第一个编号,最后一行就是最后一个编号。绝不输出解释、标题、空行、代码块标记或任何其他文字。

翻译风格:中文口语,意译优先,简短自然,贴近歌词语感。
```

与现行 prompt 的差异(迭代时注意):
- 现行版没禁竖线 `|`(协议分隔符!译文含 `|` 会截断)——A/B 版已加。
- 现行版 ad-lib 策略是"原文照抄"(`编号|原文`),但解析层会因"译文==原文"剔除 → 等效于跳过但浪费 token;A/B 统一改「编号|-」。
- `{TARGET}` 占位符 = `languageName(of:)`(`:509`)的输出;v1 实际就是简体中文。

## 解析契约(实现对照 `parseAnswer:592` + HARDENING H3/H4)

1. 逐行匹配 `编号|译文`(兼容全角 `｜`);不匹配的行(解释/```/`<think>` 泄漏)直接忽略。
2. 重复编号**末个生效**(模型先 echo 输入再输出译文时,首个生效会把原文当译文收下)。⚠️ 现实现是先到先得,H3 修时一并改。
3. `-` 与 译文==原文 → 跳过该行(不写 attachment,该行显示层自动只显原文)。
4. sanitize 见 HARDENING H4。
5. 覆盖率分层门控见 HARDENING H2。

## 成本/延迟参考

- 典型歌 30-80 行、每行 <60 字符;`行号|原文` 打包 + prompt ≈ 1.5-3k input tokens,输出 ≈ 1-2.5k。
- DeepSeek/gpt-4o-mini 价位: **每首 ≈ $0.001-0.002**;落盘缓存后二次播放 $0。
- 延迟 10-60s 可接受(歌不中断,译文到了当场上屏)。deepseek-chat 高峰期长歌可能更慢——超时上限 130s(`performRequest:565`)。
